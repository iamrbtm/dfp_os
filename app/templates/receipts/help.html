Overview

The Receipt Management system converts receipt images and PDFs into structured expense records. The workflow is: Upload → OCR → AI Extraction → Review → Approve → Expense Ledger.

1. Upload / Capture

  • Supported formats: JPG, PNG, HEIC/HEIF, PDF (including multi-page).
  • Maximum file size: configurable (default 25 MB).
  • You can drag-and-drop, use the file picker, or capture via camera on mobile devices.
  • Uploaded files are stored unchanged for audit purposes.

2. OCR Processing

Once uploaded, the receipt image is preprocessed (orientation fix, deskew, contrast enhancement) and then OCR is run to extract raw text. Supported OCR engines:

  • PaddleOCR (default) — best accuracy for printed receipts.
  • Tesseract — open-source fallback.
  • EasyOCR — alternative.

3. AI Extraction

After OCR, a local AI model (Ollama with Qwen2.5-VL by default) converts the raw text into structured data including merchant, totals, date, and line items. A mock/test mode is available by passing test as the API key.

4. Confidence Scores

Each extracted field has a confidence score from 0 to 1. Fields below the threshold (default 0.8) are flagged for review. Low-confidence fields are visually marked in the review screen.

5. Review & Edit

The review screen shows the receipt image alongside parsed metadata. You can:

  • Edit merchant name, totals, date, and other metadata.
  • Review and edit individual line items.
  • Mark items as reviewed.
  • Add or remove line items.
  • Save drafts, approve, or reject.

6. Line Item Assignment

Each line item can be assigned to:

  • Market — expense attributed to a specific vendor market.
  • Custom Job — materials for a custom order.
  • Inventory — items that become inventory stock.
  • General Business Expense — regular operating costs.
  • Personal/Excluded — not counted as business expenses.

Bulk assignment lets you assign all items at once. Split allocations are supported within line items.

7. Tax/Fee/Discount Allocation

The system can reverse-allocate receipt-level taxes, fees, discounts, tips, deposits, and rounding back to individual line items using these methods:

  • Exact — if the receipt shows per-line amounts.
  • Taxable Proportional — allocate tax across taxable items.
  • Subtotal Proportional — allocate fees/tips by line subtotal.
  • Manual — override any allocation.

8. Inventory Prompts

When a line item is marked as an inventory candidate, you will be prompted to either add it to an existing inventory item or create a new one. Inventory is never created without user confirmation.

9. Duplicate Detection

Before approval, the system checks for duplicate receipts using a weighted score based on:

  • File hash (exact match)
  • Merchant/store name similarity
  • Date/time proximity
  • Grand total match
  • Receipt number match

If flagged, you can keep, reject, or merge the duplicate.

10. Approval & Expense Records

When a receipt is approved:

  • Status changes to Approved.
  • Expense ledger records are created for each assigned line item.
  • Expenses feed into Markets, Custom Jobs, Inventory, and Analytics.
  • Double-counting is prevented.

Best Practices

  • Take photos in good lighting with the receipt flat and unwrinkled.
  • Avoid shadows and glare.
  • PDF scans are preferred for long receipts.
  • Always review AI-extracted data before approving.
  • Check the reconciliation summary to verify totals match.

Known Limitations

  • Handwritten receipts may not parse correctly.
  • OCR accuracy depends on image quality.
  • AI extraction requires Ollama to be running.
  • Real card processing is not implemented — use external payment recording.
